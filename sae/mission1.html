Mission 1

Pour cette premiere mission nous devions prendre connaissance dse données de l'entreprise Kdou afin de se familiariser avec ses dernières et de detecter de possibles erreurs dans les données afin de les en notifier. Pour ce faire nous avons utiliser le logiciel Excel, un logiciel de tableau et de feuilles de calculs.

Pour analyser les données il faut d’abord déterminer les clés primaires qui sont :

  • « N° fournisseurs », pour la table Fournisseur
  • « Réf produit », pour la table Produit
  • « Code catégorie », pour la table Catégorie
  • « Code client », pour la Table clientèle
  • « N° commande », pour la table Commande
  • Aucune clé primaire pour la table Detailcommande

Bilan

Nous avons d'abord fait une évaluation des données pour connaitre leur qualité. Certains défauts nous sont apparus dans chacune des tables:

  1. Table Fournisseur
    • Il y a des problèmes au niveau des pays qui sont écrit de différentes façon donc peut entrainer des problèmes de reconnaissance : Australia/australie, Suéde/suède, etats unis/Etats-unis/USA
    • Il y a des problèmes de numéro de téléphones qui sont parfois absents ou incomplets
  2. Table Commande
    • Ils manquent des dates pour certaines commandes
  3. Table Clientele
    • Les adresses sont parfois incomplètes
    • Certains clients ont les mêmes numéros de téléphone
    • Au niveau des villes, paris est parfois écrit avec ou sans majuscule
  4. Table Detailcommande
    • Les prix sont parfois différents pour les mêmes produits
    • Le code client FRANCK n’est pas dans la table de clientele donc la clé primaire est incorrecte
  5. Table Produit
    • Certaines quantités par unité sont absente
    • Vbeer n’a pas de prix indiqué
    • Certains produits n’ont pas d’unité de stock

Il y a donc de nombreuses erreurs et incohérences dans la base de données, qu'il faudra systématiquement prendre en compte et corriger.

Améliorations

Pour répondre a ces erreurs plusieurs solutions existent. Tout d'abord il faut passer sur le logiciel Access qui est plus efficace que Excel en terme de base de données et qui permet de mettre des contraintes d'intégrité (CI). On peut par exemple :

  1. Utiliser des CI de domaine pour limiter les erreurs concernant les adresses ou les dates. Par exemple, ne permettre que le remplissage de valeurs de type date pour les dates, ou de nombres d'une certaine longueur pour les numéros de téléphone.
  2. Utiliser une CI d’entité pour ne pas avoir de cellules vides comme pour les dates ou les numéros de téléphone, en interdisant les valeurs nulles
  3. Utiliser des CI référentielles pour obliger, lorsque certaines cases doivent être remplies, la reconnaissance d'une clé primaire dans un autre attribut, sans quoi il serait impossible d'entrer une valeur dans la case. Cela permettrait d'éviter des erreurs, comme une clé primaire contenant une faute d'orthographe
  4. Utiliser une CI déclarative pour empêché les fautes d’orthographes et les doublons de catégories en créant des ensembles contenant le nom des valeurs que l’on peut rentrer dans une catégorie, et empêchant toute autre valeur de pouvoir être remplie, par exemple un Ensemble Pays={ Australie, suède…} Pour éviter de faire rentrer des valeurs avec des différences d’orthographes
  5. Utiliser un CI déclarative pour automatiser la création de certaines clés primaires comme le numéro de fournisseur et ainsi imposer la mise en place de valeurs par défaut comme clés primaires

Ainsi, en appliquant ces modifications à l'aide de logiciels et de SGBD comme Access, il sera possible d'inclure ces contraintes d'intégrité. Cela rendra la base de données plus cohérente et plus pratique à utiliser pour les missions futures.
